# MVP: Ghost Layout Fixer

## 1. Назначение
CLI-утилита для Windows, которая помогает убрать фантомную раскладку клавиатуры, например `English (United Kingdom)`, если она отображается в переключателе языка, но отсутствует в настройках Windows или возвращается после перезагрузки.

## 2. Цель MVP
Решить один конкретный кейс:
- в системе есть фантомная раскладка;
- штатное удаление не помогает;
- после перезагрузки раскладка появляется снова;
- утилита находит следы проблемы и помогает безопасно их удалить.

## 3. Формат
- Тип: CLI-утилита
- Язык: C++
- Платформа: Windows 10/11
- Интерфейс: консольные команды

## 4. Поддерживаемый кейс первой версии
Первая версия ориентирована на:
- фантомную раскладку `en-GB`;
- пользователя с правами администратора;
- ручной запуск утилиты по команде.

## 5. Команды MVP
### `scan`
Показывает:
- какие раскладки и языковые записи обнаружены;
- найдены ли следы `en-GB`;
- в каких местах реестра найдены совпадения.

### `fix --layout en-GB`
Выполняет:
1. backup изменяемых веток реестра;
2. попытку штатного добавления проблемной раскладки;
3. попытку штатного удаления раскладки;
4. очистку известных хвостов в реестре;
5. вывод отчёта о выполненных действиях.

### `fix --layout en-GB --dry-run`
Ничего не меняет.
Только показывает:
- что будет проверено;
- какие записи будут удалены;
- какие команды будут выполнены.

### `backup`
Создаёт резервную копию затрагиваемых веток реестра в `.reg`.

### `restore --file <path>`
Восстанавливает ранее сохранённый `.reg` backup.

## 6. Какие места проверяет MVP
Минимально:
- `HKEY_CURRENT_USER\Keyboard Layout\Preload`
- `HKEY_CURRENT_USER\Keyboard Layout\Substitutes`
- `HKEY_USERS\.DEFAULT\Keyboard Layout\Preload`

## 7. Ограничения MVP
- без GUI;
- без фоновой службы;
- без автозапуска;
- без поддержки всех раскладок мира;
- без массового редактирования языковых профилей;
- без гарантии исправления всех нестандартных кейсов Windows.

## 8. Требования безопасности
- любые изменения только после явного подтверждения;
- обязательный backup перед изменениями;
- наличие режима `--dry-run`;
- лог всех выполненных действий;
- удаление только записей, связанных с выбранной раскладкой.

## 9. Критерии готовности MVP
MVP считается готовым, если:
1. утилита собирается и запускается на Windows;
2. команда `scan` показывает найденные следы;
3. команда `fix --layout en-GB --dry-run` корректно показывает план действий;
4. команда `fix --layout en-GB` создаёт backup и выполняет очистку;
5. после применения фикса и перезагрузки фантомная раскладка в целевом кейсе не возвращается;

## 10. Что можно добавить после MVP
- поддержка других раскладок;
- экспорт лога;
- более глубокий поиск следов в профиле пользователя;
- создать батник, который сразу находит призрачные языки, делает резервную копию, чистит реестр
